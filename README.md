
# Estructura del Proyecto de Portafolio Personal

A continuación se presenta la estructura base del repositorio y sitio web, diseñada para funcionar perfectamente con GitHub Pages, permitir escalabilidad y mantener un flujo ordenado:

```
/mi-portafolio/
│── index.html
│── about.html
│── projects.html
│── blog.html
│── 404.html
│── CNAME
│── README.md
│
├── /assets/
│   ├── /css/
│   │     styles.css
│   │     theme.css
│   │
│   ├── /js/
│   │     main.js
│   │     analytics.js
│   │
│   ├── /img/
│   │     /perfil/
│   │     /proyectos/
│   │     /general/
│   │
│   └── /fonts/
│
├── /projects/
│   └── nombre-proyecto/
│          index.md
│          assets/
│          executable/
│          notas.md
│
├── /blog/
│   ├── _posts/
│   ├── tags/
│   └── categories/
│
└── /data/
    proyectos.json
    info.json
```

# README.md — Versión Inicial

## Portafolio Personal de Juan

Este repositorio contiene el desarrollo del portafolio personal donde se documentan proyectos, investigaciones, herramientas de software libre y contenido técnico. El sitio será publicado mediante  **GitHub Pages** .

---

## Contenido del Sitio

### **Inicio**

Presentación general y acceso rápido a las secciones principales.

### **Proyectos**

Catálogo de proyectos destacados, cada uno con:

* Descripción técnica
* Funcionalidades principales
* Enlaces al repositorio en GitHub
* Ejecutables y releases
* Documentación complementaria

### **Blog**

Espacio para artículos técnicos, análisis, investigaciones, documentación extendida y notas profesionales. Construido con soporte para  **Jekyll** , permitiendo publicaciones en formato Markdown.

### **Acerca de mí**

Hoja de vida profesional en formato web, con experiencia, habilidades, certificaciones e información académica.

---

## Tecnologías utilizadas

* GitHub Pages
* HTML, CSS y JavaScript
* Jekyll para la sección del blog
* Archivos JSON para estructurar datos dinámicos

---

## Estructura del Repositorio

La estructura del proyecto está diseñada para ser escalable, organizada y mantenible. Incluye carpetas para estilos, scripts, imágenes, proyectos, blog y datos.
